# 画面プレビューの見方

このフォルダは、病院予約カレンダーアプリの画面構成と操作の流れを確認するためのプレビュー集です。  
患者として使う場合の流れと、管理者として使う場合の流れをこの 1 ファイルで追えるようにまとめています。

## 1. 利用者の種類

### 未ログイン利用者

- ログインや新規登録を行う前の状態です
- 予約はまだできません
- `ログイン` または `新規登録` へ進みます

### 通常ユーザー

- 予約の登録、確認、変更、キャンセルができます
- プロフィールの確認と更新ができます

### 管理者ユーザー

- 通常ユーザーの機能に加えて、管理者画面へ入れます
- 診療科管理、休診日管理、予約確認、監査ログ確認ができます

## 2. 画面一覧

### ホーム（未ログイン）

画像: [ホーム.png](./ホーム.png)

この画面でできること:

- ログイン画面へ進む
- 新規登録画面へ進む
- このアプリで何ができるかを確認する

画面の意味:

- 予約を始める前の案内画面です

画像: [通常ユーザー.png](./通常ユーザー.png)

- 診療科を選んで新しい予約へ進む

画面の意味:

- 通常ユーザーの入口画面です
- 一番よく使う操作をここに集めています

### ホーム（管理者権限ありユーザー）

画像: [管理者権限ありユーザー.png](./管理者権限ありユーザー.png)

### 新しい予約

画像: [新しい予約.png](./新しい予約.png)

この画面でできること:

- カレンダーから受診日を選ぶ
- 予約可能時間を確認する

画面の意味:

- 診療科ごとの空き状況を日付単位で確認する画面です

### 予約一覧

画像: [予約一覧.png](./予約一覧.png)

画面の意味:

- 自分が持っている予約をまとめて確認する画面です

### 予約詳細

画像: [予約詳細.png](./予約詳細.png)

この画面でできること:

- 予約内容を確認する
- 日時変更へ進む
- 予約をキャンセルする

画面の意味:

- 1 件の予約を詳しく確認する画面です
- 当日の案内もここに表示します

### 診療科管理

- 一覧から直接編集できるようにして、追加と並び替えを分かりやすくしています

## 3. 患者向け画面フロー

### 初回利用の流れ

1. `ホーム（未ログイン）`
2. `新規登録`
3. `ホーム（通常ユーザー）`

### 予約を取る流れ

1. `ホーム（通常ユーザー）`
2. 診療科を選ぶ
3. `新しい予約`
4. 日付を選ぶ
5. 時間を選ぶ
6. `予約一覧`
7. 必要なら `予約詳細`

### 予約を変更する流れ

1. `予約一覧`
2. 対象の予約を選ぶ
3. `予約詳細`
4. `日時変更`
5. 変更完了後に `予約詳細`

### 予約をキャンセルする流れ

1. `予約一覧`
2. または `予約詳細`
3. キャンセル実行
4. `予約一覧` に戻る

## 4. 管理者向け画面フロー

### 管理画面へ入る流れ

1. 管理者権限ありユーザーでログイン
2. `ホーム（管理者権限ありユーザー）`
3. `管理者画面`

### 診療科を管理する流れ

1. `管理者画面`
2. `診療科管理ページを開く`
3. `診療科管理`
4. 追加、名前変更、並び替え、受付切替、削除を行う

### 休診日を管理する流れ

1. `管理者画面`
2. 診療科と日付を選ぶ
3. 休診日を登録する
4. 必要なら登録済み一覧から解除する

### 日付ごとの予約を確認する流れ

1. `管理者画面`
2. 診療科と日付を選ぶ
3. 予約一覧を確認する
4. 必要なら代行キャンセルする

## 5. 主要な操作の意味

### 予約する

- 利用者が診療科と日時を選び、予約データを作成する操作です
- 同じ診療科の重複予約や同じ時間の重複予約はできません

### 日時を変更する

- すでにある予約の日時だけを変更する操作です
- 診療科はそのままで、空いている時間へ動かします

### キャンセルする

- 予約データを削除する操作です
- キャンセル後は予約一覧から消えます

### 受付状態を切り替える

- 管理者が診療科の新規受付を止めたり再開したりする操作です
- 既存予約は自動では消えません

### 休診日を登録する

- 特定の診療科と日付に対して、新しい予約を受け付けないようにする操作です

## 6. 補足

- 通知機能はまだ未実装です
- 管理者も通常の患者機能を使えます
- 診療科の並び順は診療科管理ページで変更できます
- 予約データ、プロフィール、診療科情報は Firebase に保存されます

## 7. このフォルダのおすすめの読み方

1. まず `ホーム.png` と `新規登録.png` を見る
2. 次に `通常ユーザー.png`、`新しい予約.png`、`予約一覧.png`、`予約詳細.png` を追う
3. 最後に `管理者画面.png` と `診療科管理.png` を見る

この順番で見ると、患者向けの操作から管理者向けの操作まで流れで理解しやすくなります。
