## 1. 前提

この計画は、現在の React + Firebase 実装を基準にする。

- 総合病院で複数診療科がある
- 利用者は 10 代から 80 代以上
- 予約の登録、変更、削除、閲覧を行う
- アプリ本体は Firebase Authentication / Cloud Firestore を使う

---

## 2. 実装方針

- 永続化する `Slot` テーブルは持たない
- 予約可能時間は診療ルールから動的生成する
- Firestore トランザクションで予約とロック文書を同時更新する
- 利用者名は `user_name` に統一する
