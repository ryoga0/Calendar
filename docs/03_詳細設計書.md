## 1. 概要

本書は、Firebase 完全移行後の患者向け予約機能と、初期版の管理者機能の内部仕様を定義する。  
現在は React から Firebase Authentication / Cloud Firestore を直接利用し、予約更新系は Firestore トランザクションで処理する。

## 3. フロントエンド構成

### 3.1 主要ファイル

| ファイル | 役割 |
|----------|------|
| `src/auth/AuthContext.jsx` | Firebase セッション監視、現在ユーザー保持 |
| `src/firebase/client.js` | Firebase 初期化と session persistence |
| `src/firebase/errors.js` | Firebase エラーの日本語化 |
| `src/firebase/hospitalSchedule.js` | 診療時間ルールと時刻計算 |
| `src/firebase/patientPortal.js` | 認証、プロフィール、診療科、空き状況、予約処理、管理者処理 |
| `src/pages/AdminDashboard.jsx` | 管理者画面 |
| `src/api/*.js` | 画面側から使う薄いラッパー |

### 3.2 セッション管理

- Firebase Auth の `browserSessionPersistence` を使う
- `onIdTokenChanged` でログイン状態を監視する
- ID トークンは `AuthContext` で保持する
- リロード時は Firebase Auth のセッションから復元する

## 4. Firestore データ設計

## 4.1 `users/{uid}`

| フィールド | 型 | 内容 |
|------------|----|------|
| `email` | string | メールアドレス |
| `user_name` | string | 利用者名 |
| `phone` | string/null | 電話番号 |
| `created_at` | string | ISO 8601 |
| `updated_at` | string | ISO 8601 |

## 4.2 `departments/{departmentId}`

| フィールド | 型 | 内容 |
|------------|----|------|
| `name` | string | 診療科名 |
| `sort_order` | number | 表示順 |
| `is_active` | boolean | 予約受付中か |
| `created_at` | string | 任意 |
| `updated_at` | string | 任意 |

## 4.3 `admins/{uid}`

| フィールド | 型 | 内容 |
|------------|----|------|
| `email` | string | 任意。管理対象ユーザーの識別用 |
| `created_at` | string | 任意 |
| `updated_at` | string | 任意 |

## 4.4 `department_closures/{departmentId}_{YYYY-MM-DD}`

| フィールド | 型 | 内容 |
|------------|----|------|
| `department_id` | string | 診療科 ID |
| `department_name` | string | 診療科名スナップショット |
| `date` | string | `YYYY-MM-DD` |
| `reason` | string/null | 休診理由 |
| `created_by` | string | 登録した管理者 UID |
| `created_at` | string | ISO 8601 |
| `updated_at` | string | ISO 8601 |

## 4.5 `users/{uid}/appointments/{appointmentId}`

| フィールド | 型 | 内容 |
|------------|----|------|
| `user_id` | string | Firebase UID |
| `department_id` | string | 診療科 ID |
| `department_name` | string | 診療科名スナップショット |
| `status` | string | 現在は `confirmed` のみ |
| `slot_key` | string | `YYYYMMDDTHHMM` |
| `start_at` | string | `YYYY-MM-DDTHH:mm:00` |
| `created_at` | string | ISO 8601 |
| `updated_at` | string | ISO 8601 |

## 4.6 ロックコレクション

### `user_department_locks/{uid}_{departmentId}`

- 同一診療科の重複予約防止

### `user_slot_locks/{uid}_{slotKey}`

- 同時刻の重複予約防止

### `department_slot_locks/{departmentId}_{slotKey}`

- 診療科 × 時刻の満員判定

## 4.7 `audit_logs/{logId}`

| フィールド | 型 | 内容 |
|------------|----|------|
| `action` | string | 操作種別 |
| `actor_type` | string | `patient` / `admin` |
| `actor_user_id` | string | 実行者 UID |
| `actor_user_name` | string/null | 実行者名 |
| `actor_email` | string/null | 実行者メールアドレス |
| `target_type` | string | 操作対象種別 |
| `target_id` | string | 操作対象 ID |
| `summary` | string | 画面表示用の要約 |
| `details` | map | 詳細情報 |
| `created_at` | string | ISO 8601 |

## 5. 診療時間仕様

| 曜日 | 時間帯 |
|------|--------|
| 平日 | 09:00 / 10:00 / 11:00 / 13:00 / 14:00 / 15:00 / 16:00 |
| 土曜 | 09:00 / 10:00 / 11:00 |
| 日曜 | なし |

補足:

- 時刻比較は `Asia/Tokyo` 前提
- 現在時刻以前は `受付終了`
- `department_slot_locks` が存在する時刻は `満員`
- `department_closures` が存在する日付は `休診日`

## 6. 認証仕様

### 6.1 新規登録

1. Firebase Authentication でユーザー作成
2. `displayName` に `user_name` を反映
3. `users/{uid}` を upsert
4. ID トークンとプロフィールを `AuthContext` に反映

### 6.2 ログイン

1. Firebase Authentication でサインイン
2. `users/{uid}` を補完
3. `AuthContext` に反映

### 6.3 ログアウト

1. Firebase Authentication からサインアウト
2. `AuthContext` の `token`, `user` をクリア

## 7. 予約仕様

## 7.1 空き状況取得

入力:

- `departmentId`
- `date`
- `excludeAppointmentId` 任意

出力:

```json
{
  "department_id": "department-id",
  "date": "2026-04-10",
  "closed": false,
  "closed_reason": null,
  "items": [
    {
      "time": "09:00",
      "start_at": "2026-04-10T09:00:00",
      "available": true,
      "reason": null
    }
  ]
}
```

## 7.2 予約作成

トランザクション内で次を実行する。

1. 診療科存在確認
2. 診療時間内・未来時刻であることを確認
3. `department_closures` の存在確認
4. `user_department_locks` の存在確認
5. `user_slot_locks` の存在確認
6. `department_slot_locks` の存在確認
7. `appointments` 文書作成
8. 3 種類のロック文書を作成

## 7.3 予約変更

トランザクション内で次を実行する。

1. 対象予約存在確認
2. 同一診療科のまま新しい日時を検証
3. `department_closures` を確認
4. 新しい `user_slot_locks` を確認
5. 新しい `department_slot_locks` を確認
6. 予約文書を更新
7. 古い時刻のロックを削除
8. 新しい時刻のロックを作成

## 7.4 予約削除

トランザクション内で次を実行する。

1. 対象予約存在確認
2. 予約文書削除
3. 3 種類のロック文書削除

## 7.5 管理者機能

### 管理者判定

- `admins/{uid}` 文書が存在する利用者を管理者として扱う
- 一般利用者が自分で管理者権限を付ける UI はまだ持たない

### 診療科管理

- `departments/{departmentId}.is_active` を切り替える
- `false` の診療科は患者画面の一覧から除外する

### 休診日管理

- `department_closures` に文書を作成 / 削除する
- 患者画面の空き状況では、該当日をすべて `休診日` として表示する

### 予約運用

- 管理者は日付と診療科を指定して予約一覧を確認できる
- 管理者は予約を代行キャンセルできる
- 主要な更新操作は `audit_logs` に記録する
- 通知機能は未実装のため、代行キャンセル時に通知は送られない

## 8. UI とエラー表示

- 業務エラーは画面上部 Alert に表示する
- Firebase SDK の生エラーは `src/firebase/errors.js` で日本語に変換する

代表例:

| 状況 | 表示文言 |
|------|----------|
| 重複診療科 | この診療科はすでに予約済みです。変更する場合は予約一覧からお進みください。 |
| 同時刻競合 | 同じ時間に別の予約があります。別の日時をお選びください。 |
| 満員 | この日時は満員です。別の日時をお選びください。 |
| 過去時刻 | この日時は予約できません。別の日時をお選びください。 |
| 同日時再指定 | 同じ日時は選択できません。別の日時をお選びください。 |
