## 1. 概要

本書は患者向けおよび**管理画面**の主要シナリオにおける**画面遷移**と**分岐**を Mermaid で表現する。画面IDは基本設計書の画面一覧（SC-／AD-）に対応する。

---

## 2. 全体ナビゲーション（サイトマップ）

```mermaid
flowchart TB
  subgraph public [未ログイン]
    SC01[SC-01 ログイン]
    SC02[SC-02 新規登録]
  end
  subgraph auth [ログイン後]
    SC03[SC-03 ホーム]
    SC04[SC-04 診療科選択]
    SC05[SC-05 日付・枠選択]
    SC06[SC-06 予約確認]
    SC07[SC-07 予約一覧]
    SC08[SC-08 予約詳細]
    SC09[SC-09 予約変更]
    SC10[SC-10 プロフィール]
  end
  SC01 --> SC02
  SC02 --> SC01
  SC01 --> SC03
  SC03 --> SC04
  SC03 --> SC07
  SC03 --> SC10
  SC04 --> SC05 --> SC06 --> SC07
  SC07 --> SC08
  SC08 --> SC09
  SC08 --> SC07
  SC09 --> SC08
```

## 3. 新規登録〜初回ログイン

```mermaid
flowchart TD
  A[アプリ起動] --> B{ログイン済み?}
  B -->|Yes| H[SC-03 ホーム]
  B -->|No| C[SC-01 ログイン]
  C --> D{新規利用?}
  D -->|Yes| E[SC-02 新規登録]
  E --> F{登録成功?}
  F -->|No| E
  F -->|Yes| G[自動ログイン or SC-01へ]
  G --> H
  D -->|No| I{認証成功?}
  I -->|No| C
  I -->|Yes| H
```

## 4. 予約登録フロー（閲覧の前提となる空き確認を含む）

初診・再診で分岐しない。

- **選択した診療科に、すでに確定予約がある**場合は新規不可（同一診療科は同時1件まで）。
- **確定予約が既に3件**（＝異なる診療科3件）のときは、4件目を拒否。

```mermaid
flowchart TD
  Start[SC-03 ホーム] --> A[予約する]
  A --> LIM3{確定予約は3件未満?}
  LIM3 -->|No| MSG3[3診療科上限メッセージ]
  LIM3 -->|Yes| B[SC-04 診療科選択]
  B --> DUP{この診療科に確定済み?}
  DUP -->|Yes| MSG1[同一科1件までメッセージ]
  DUP -->|No| C[SC-05 日付・枠選択 1時間枠]
  C --> D{空き枠あり?}
  D -->|No| C
  D -->|Yes| E[枠を選択]
  E --> F[SC-06 予約確認]
  F --> G{確定?}
  G -->|戻る| C
  G -->|確定実行| H{API 成功?}
  H -->|409 競合等| I[エラー表示]
  I --> C
  H -->|400 同一科あり| MSG1
  H -->|400 3件上限| MSG3
  H -->|成功| J[SC-07 予約一覧]
```

## 5. 予約閲覧フロー

```mermaid
flowchart TD
  H[SC-03 ホーム] --> L[SC-07 予約一覧]
  L --> M{件数}
  M -->|0件| N[空状態メッセージ + 予約へ誘導]
  N --> B[SC-04 診療科選択]
  M -->|1件以上| O[カード選択]
  O --> P[SC-08 予約詳細]
```

---

## 6. 予約変更フロー

**診療科は変更しない。日時（1時間枠）のみ**選び直す。

```mermaid
flowchart TD
  P[SC-08 予約詳細] --> Q{変更可能? 締切・状態}
  Q -->|不可| R[理由を表示 例: 締切超過]
  Q -->|可| S[SC-09 予約変更]
  S --> U[同一診療科の 1時間枠を選択]
  U --> V[変更内容確認]
  V --> W{更新実行}
  W -->|成功| P
  W -->|失敗| U
```

---

## 7. 予約削除（キャンセル）フロー

```mermaid
flowchart TD
  P[SC-08 予約詳細] --> X[キャンセル]
  X --> Y[確認ダイアログ]
  Y --> Z{確定?}
  Z -->|いいえ| P
  Z -->|はい| AA{API 成功?}
  AA -->|No| AB[エラー表示]
  AB --> P
  AA -->|Yes| AC[ステータス更新表示]
  AC --> AD[SC-07 予約一覧]
```

---

## 8. プロフィール編集フロー

```mermaid
flowchart TD
  H[SC-03 ホーム] --> PR[SC-10 プロフィール]
  PR --> ED[編集]
  ED --> SV[保存]
  SV --> OK{成功?}
  OK -->|Yes| PR
  OK -->|No| ER[エラー表示]
  ER --> ED
  PR --> LG[ログアウト]
  LG --> SC01[SC-01 ログイン]
```

## 10. 管理：休診登録フロー

```mermaid
flowchart TD
  AD02[AD-02 ダッシュボード] --> AD06[AD-06 休診カレンダー]
  AD06 --> SEL[診療科と日付を選択]
  SEL --> WARN{同日に予約あり?}
  WARN -->|Yes| NOTE[警告表示 AD-07へ誘導]
  WARN -->|No| OK[登録確定]
  NOTE --> AD07[AD-07 で代行キャンセル等]
  OK --> AD06
```

  TOG --> SAVE[保存]
